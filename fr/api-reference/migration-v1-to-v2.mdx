---
title: Migration — Integrations vers Channels
icon: arrows-rotate
---

L'API REST publique utilise desormais **channels** (canaux) a la place d'**integrations** (integrations) — alignee avec l'UI du tableau de bord et la couche MCP. Les anciennes routes continuent de fonctionner et seront retirees le **13/11/2026**.

## Pourquoi ce changement

Le tableau de bord Postzee, le serveur MCP, la skill Claude Code et le noeud n8n utilisent deja le mot **channel** pour decrire un compte social connecte. L'API REST publique etait la derniere surface qui utilisait encore le terme legacy **integration**. Cette migration aligne tous les consommateurs autour d'un vocabulaire coherent.

## Fenetre de depreciation

| Date | Evenement |
|------|-----------|
| 13/05/2026 | `/public/v1/channels` publie. `/public/v1/integrations` marque comme deprecie. |
| 13/11/2026 | `/public/v1/integrations` legacy est **supprime**. Les appels retournent `410 Gone`. |

Pendant la fenetre, les appels legacy fonctionnent normalement, mais retournent :

```
Deprecation: true
Sunset: Fri, 13 Nov 2026 00:00:00 GMT
Link: </public/v1/channels>; rel="successor-version"
Warning: 299 - "Deprecated API — use /public/v1/channels"
```

## Ce qui a change

### 1. Endpoint de listage

| Avant | Apres |
|-------|-------|
| `GET /public/v1/integrations` | `GET /public/v1/channels` |

### 2. Nom du champ dans la reponse

| Avant (`identifier`) | Apres (`platform`) |
|---|---|
| `"identifier": "instagram"` | `"platform": "instagram"` |

Les valeurs elles-memes n'ont pas change — memes identifiants (`instagram`, `linkedin-page`, `tiktok`, `x`, etc.). Seul le nom du champ a change.

### 3. Format du corps de creation de post

| Avant | Apres |
|-------|-------|
| `posts[].integration.id` | `posts[].channel.id` |

```diff
  {
    "type": "schedule",
    "date": "2026-05-01T14:00:00Z",
    "posts": [
      {
-       "integration": { "id": "channel-uuid" },
+       "channel": { "id": "channel-uuid" },
        "value": [ ... ]
      }
    ]
  }
```

<Warning>
Envoyer **a la fois** `channel` et `integration` dans le meme objet de post retourne `400 Bad Request`. Utilisez l'un ou l'autre.
</Warning>

### 4. Nouveau endpoint — obtenir un canal specifique

```
GET /public/v1/channels/:id
```

L'API precedente n'exposait pas cette route. Utile pour rafraichir les metadonnees d'un canal sans payer le cout d'un listage complet.

## Ce qui n'a PAS change

- Les valeurs des identifiants de plateforme (`instagram`, `tiktok`, etc.)
- L'authentification (header `Authorization`, meme cle API)
- Les rate limits
- La reponse de `GET /public/v1/posts` — inclut encore l'objet legacy `integration` (avec `identifier`). Cela changera dans une future version majeure.
- Tous les autres endpoints (`/posts`, `/media`, `/credits`, `/jobs`, `/find-slot`)

## Etapes de migration

### Si vous appelez l'API REST directement

1. Remplacez `GET /public/v1/integrations` par `GET /public/v1/channels`.
2. Dans le parsing de la reponse, renommez `identifier` → `platform`.
3. Dans le corps de `POST /public/v1/posts`, renommez `integration` → `channel` dans chaque objet de post.

### Si vous utilisez le noeud personnalise n8n

Mettez `n8n-nodes-postzee` a jour vers la version **2.0.0** ou superieure. Aucun changement de workflow requis — le noeud gere la traduction en interne. Les workflows enregistres continuent de fonctionner apres la mise a jour.

### Si vous utilisez le serveur MCP Postzee ou la skill Claude Code

Aucune action requise. Les deux utilisent deja la terminologie moderne `channel` / `platform`.

## Exemples de code

### Avant (legacy)

```bash
curl -H "Authorization: YOUR_API_KEY" \
  https://api.postzee.app/public/v1/integrations

# Reponse : [{ "id": "...", "identifier": "instagram", ... }]

curl -X POST -H "Authorization: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "now",
    "posts": [{
      "integration": { "id": "channel-uuid" },
      "value": [{ "content": "Bonjour", "id": "uuid", "image": [] }]
    }]
  }' \
  https://api.postzee.app/public/v1/posts
```

### Apres (moderne)

```bash
curl -H "Authorization: YOUR_API_KEY" \
  https://api.postzee.app/public/v1/channels

# Reponse : [{ "id": "...", "platform": "instagram", ... }]

curl -X POST -H "Authorization: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "now",
    "posts": [{
      "channel": { "id": "channel-uuid" },
      "value": [{ "content": "Bonjour", "id": "uuid", "image": [] }]
    }]
  }' \
  https://api.postzee.app/public/v1/posts
```
