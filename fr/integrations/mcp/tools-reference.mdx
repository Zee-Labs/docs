Reference complete des **25 outils** exposes par le serveur MCP de Postzee. Ces outils sont disponibles pour tout agent IA compatible MCP (Claude Code, OpenClaw, Hermes Agent ou clients MCP directs).

<Note>
Tous les outils acceptent la cle API integree dans votre URL MCP. Ils sont limites a une seule organisation. **Les couts de generation IA sont toujours retournes en credits, jamais en dollars.** ($1 USD = 1 000 credits.)
</Note>

## Modeles et Specs

### POSTZEE_LIST_MODELS_DETAILED

Matrice de capacites pour tous les modeles d'image et video disponibles ā€” durees, resolutions, capacites audio, parametres customises acceptes, et tier de cout relatif.

**Parametres :**

| Parametre | Type | Requis | Description |
|-----------|------|:---:|-------------|
| `type` | string | Non | `"image"`, `"video"` ou `"all"` (default) |
| `includeHeygen` | boolean | Non | Inclut l'entree synthetique HeyGen. Default : `true` |
| `family` | string | Non | Filtre vers une seule famille (ex. : `"ideogram-v3"`, `"sora-2"`, `"veo3.1"`) |
| `agenticDefault` | boolean | Non | Quand `true`, ne retourne que les entrees marquees comme choix recommande pour l'automatisation |

**Retourne (par modele) :**
```json
{

  "supportsImg2Img": false,
  "supportsImg2Vid": false,
  "supportsVector": false,
  "durations": [10, 15],
  "resolutions": ["1080p"],
  "aspectRatios": ["16:9", "9:16"],
  "recommendedAspectsByPlatform": {
    "instagram_reels": "9:16",
    "tiktok_feed": "9:16",
    "youtube_feed": "16:9"
  },
  "customParamsAccepted": {
    "duration": { "type": "enum", "enum": [10, 15], "description": "..." }
```

### POSTZEE_VALIDATE_GENERATION

Validation pre-execution ā€” capture les erreurs de parametres, les credits insuffisants, les problemes de stockage et les blocages de forfait **avant** de bruler des credits.

**Parametres :**

| Parametre | Type | Requis | Description |
|-----------|------|:---:|-------------|
| `type` | string | Oui | `"image"` ou `"video"` |
| `model` | string | Oui | ID du modele |
| `duration` | number | Non | Secondes |
| `aspectRatio` | string | Non | ex. : `"4:5"` |
| `resolution` | string | Non | Resolution de sortie |
| `quality` | string | Non | Sora Pro `"standard"` / `"high"` |
| `generateAudio` | boolean | Non | |
| `voiceControl` | boolean | Non | |
| `imageUrl` | string | Non | Requis pour les modeles image-to-video |
| `slideCount` | number | Non | Multiplicateur de batch (ex. N variations du meme prompt). **Pas pour les carrousels** ā€” ceux-ci passent par `POSTZEE_RENDER_CAROUSEL` avec son propre modele de cout. |

**Retourne :**
```json
{
  "valid": true,
  "errors": [],
  "warnings": ["Storage at 85% ā€” approaching limit."],
  "estimatedCreditsPerItem": 160,
  "totalEstimatedCredits": 1600,
  "slideCount": 10,
  "currentBalance": 4500,
  "willExceedBalance": false,
  "shortfall": 0
}
```

Quand `valid: false`, le tableau `errors[]` contient des problemes lisibles ā€” ex. : duree pas dans l'ensemble autorise par le modele, `imageUrl` manquant pour modele i2v, stockage a 100%, ou modele enregistre mais pas encore disponible via MCP.

## Canaux et Credits

### POSTZEE_LIST_CHANNELS

Liste les canaux de reseaux sociaux connectes avec des flags d'action explicites.

**Parametres :** Aucun

**Retourne :**
```json
[
  {
    "id": "channel-uuid",
    "name": "My Instagram",
    "platform": "instagram",
    "picture": "https://...",
    "profile": "https://...",
    "disabled": false,
    "requiresReauth": false,
    "canPostNow": true,
    "actionRequired": "none",
    "actionUrl": null,
    "actionMessage": null,
    "status": "active",
    "statusMessage": null
  }
]
```

`actionRequired` est l'un de `"none"` (pret a publier), `"reconnect"` (token expire ā€” l'utilisateur reconnecte) ou `"reenable"` (admin/billing ā€” l'utilisateur reactive, **pas** la meme chose que reconnecter).

`statusMessage` est une explication lisible quand le canal necessite une action (ex : `"Token du canal expire. Reconnectez pour continuer a publier."`). Il est `null` quand `actionRequired` est `"none"`.

## Memoire des Medias

Les deux outils ci-dessous permettent a un agent de retrouver et reutiliser des actifs medias ā€” generes par IA ou televerses par l'utilisateur ā€” entre tours et entre sessions, sans redemander les URLs a l'utilisateur.

### POSTZEE_LIST_MEDIA

Liste des medias recents, limitee a l'organisation. Renvoie les plus recents d'abord.

**Parametres :**

| Parametre | Type | Requis | Description |
|-----------|------|:---:|-------------|
| `type` | string | Non | `"image"`, `"video"` ou `"all"` (par defaut) |
| `source` | string | Non | `"generated"` (cree par IA), `"uploaded"` (importe via `POSTZEE_UPLOAD_MEDIA`) ou `"all"` (par defaut) |
| `search` | string | Non | Recherche insensible a la casse dans le prompt original (medias IA) et le nom de fichier |
| `since` | string | Non | Date ISO ā€” ne renvoie que les medias crees apres cet instant |
| `limit` | number | Non | Par defaut 20, maximum 100 |

**Renvoie (par element) :**
```json
{
  "mediaId": "uuid",
  "url": "https://cdn.postzee.app/abc.jpg",
  "name": "abc.jpg",
  "type": "image",
  "source": "generated",
  "prompt": "chien sur la plage au coucher du soleil",
  "modelUsed": "ideogram-v3-turbo",
  "createdAt": "2026-05-07T14:23:00Z",
  "sizeBytes": 245678,
  "dimensions": { "width": 1024, "height": 1280 }
}
```

**Renvoie (echec) :**
```json
{ "error": "list_media_failed", "message": "..." }
```

<Note>
Les medias avec soft-delete sont exclus automatiquement. Les medias utilises comme photo de profil sont egalement exclus.
</Note>

---

### POSTZEE_UPLOAD_MEDIA

Importe une URL publique vers le storage Postzee. Renvoie un `mediaId` stable et une URL CDN reutilisable dans `GENERATE_*`, `CREATE_POST` et `LIST_MEDIA`.

**Parametres :**

| Parametre | Type | Requis | Description |
|-----------|------|:---:|-------------|
| `url` | string | Oui | URL publique `http://` ou `https://`. Doit etre accessible sans authentification. |
| `description` | string | Non | Etiquette courte enregistree avec le media (aide l'agent a tenir son manifest) |

**Renvoie (succes) :**
```json
{
  "success": true,
  "mediaId": "uuid",
  "url": "https://cdn.postzee.app/uploads/xyz.jpg",
  "type": "image",
  "sizeBytes": 524288,
  "dimensions": { "width": 1080, "height": 1920 }
}
```

**Renvoie (echec) :**
```json
{
  "success": false,
  "error": "blocked_url | unsupported_type | size_limit_exceeded | fetch_failed | storage_full | upload_failed | save_failed",
  "message": "..."
}
```

**Ordre de validation (chaque etape rejette avant la suivante) :**

1. URL parsable et utilisant `http`/`https`
2. Hostname **non** localhost / IP privee / endpoint cloud-metadata (defense SSRF)
3. Sondage HEAD reussi (ou GET ranged en fallback)
4. Content-Type correspond a l'allowlist configuree (par defaut : `image/jpeg`, `image/png`, `image/webp`, `image/gif` / `video/mp4`, `video/quicktime`, `video/webm`)
5. Taille dans le cap par type (par defaut : 25 MB image, 500 MB video)
6. Quota de storage de l'organisation a de la place

**Limites par defaut (configurables via SystemConfig) :**

| Type | Taille max | Formats acceptes |
|------|-----------|------------------|
| Image | 25 MB | JPG, PNG, WebP, GIF |
| Video | 500 MB | MP4, MOV, WebM |

Les limites et formats acceptes peuvent etre ajustes a l'execution en modifiant les cles `UPLOAD_*` dans `SystemConfig`. Cet outil MCP et les endpoints REST d'upload partagent la meme source de verite ā€” modifie une fois, s'applique partout en ~5 minutes (TTL du cache).

<Warning>
Pour la confidentialite et la securite, l'outil n'accepte jamais d'en-tetes d'authentification ni de tokens. Les URLs qui requierent un Bot Token (ex. URLs `getFile` Telegram) doivent etre re-hebergees par le client de chat sur une URL publique au prealable. Pour les uploads en base64 / fichiers directs, utilisez l'endpoint REST existant `/media/upload-server` ā€” l'outil MCP gere uniquement les imports via URL.
</Warning>

<Note>
Apres un import reussi, le media apparait dans `POSTZEE_LIST_MEDIA` avec `source: "uploaded"`.
</Note>

## Generation

### POSTZEE_GENERATE_IMAGE

Genere une image IA a partir d'un prompt. **Operation asynchrone** ā€” retourne un job id a poll avec `POSTZEE_CHECK_JOB`.

**Parametres :**

| Parametre | Type | Requis | Description |
|-----------|------|:---:|-------------|
| `prompt` | string | Oui | Description detaillee de l'image |
| `model` | string | Oui | ID du modele depuis `POSTZEE_LIST_MODELS_DETAILED` (type `"image"`) |
| `aspectRatio` | string | Non | ex. : `"1:1"`, `"4:5"`, `"9:16"`, `"16:9"`. Default : `"1:1"` |
| `imageUrls` | string[] | Non | URLs d'images de reference pour image-to-image |
| `quality` | string | Non | GPT Image 2 : `"low"`, `"medium"` ou `"high"`. Default : `"high"` |
| `style` | string | Non | Recraft V3/V4 : `"realistic_image"`, `"digital_illustration"`, `"vector_illustration"` |
| `renderingSpeed` | string | Non | Tier Ideogram V3 : `"turbo"`, `"balanced"` ou `"quality"` |
| `negativePrompt` | string | Non | Choses a exclure de l'image |
| `seed` | number | Non | Seed deterministe optionnel |

**Retourne (succes) :**
```json
{ "success": true, "jobId": "job-uuid", "status": "processing", "message": "Image generation started..." }
```

**Retourne (echec) :**
```json
{ "success": false, "error": "unsupported_model | unknown_model | wrong_type | organization_not_found | generation_failed", "message": "...", "suggestions": [{ "modelId": "...", "reason": "..." }] }
```

### POSTZEE_GENERATE_VIDEO

Genere une video IA a partir d'un prompt ou d'une image de reference. **Operation asynchrone.**

**Parametres :**

| Parametre | Type | Requis | Description |
|-----------|------|:---:|-------------|
| `prompt` | string | Oui | Description detaillee de la video |
| `model` | string | Oui | ID du modele depuis `POSTZEE_LIST_MODELS_DETAILED` (type `"video"`) |
| `duration` | number | Non | Secondes. Valeurs autorisees varient par modele ā€” consultez `model.durations` |
| `aspectRatio` | string | Non | `"16:9"` ou `"9:16"`. Default : `"16:9"` |
| `imageUrl` | string | Non | URL de l'image de reference pour modeles image-to-video |
| `endImageUrl` | string | Non | URL du frame final pour interpolation first-last-frame (ex. : `wan-flf2v`) |
| `resolution` | string | Non | `"480p"`, `"540p"`, `"720p"`, `"1080p"`, `"4k"` (depend du modele) |
| `quality` | string | Non | Sora 2 Pro uniquement : `"standard"` (720p) ou `"high"` (1080p) |
| `generateAudio` | boolean | Non | Genere de l'audio ambiant (Veo 3.1, Pixverse, Wan 2.5). Affecte les credits. |
| `voiceControl` | boolean | Non | Controle de voix (Kling 2.6/3.0 Pro). Affecte les credits. |
| `negativePrompt` | string | Non | Choses a eviter dans la video |
| `seed` | number | Non | Seed deterministe optionnel |

**Retourne :** meme structure que `POSTZEE_GENERATE_IMAGE`.

<Note>
Les IDs specifiques de tier retournes par `POSTZEE_LIST_MODELS_DETAILED` (ex. : `ideogram-v3-turbo`, `gpt-image-2-high`, `sora-2-t2v-pro-1080p`, `recraft-v4-vector`) peuvent etre passes directement dans `model`. Le MCP traduit vers la payload correcte du backend automatiquement ā€” aucun parametre `tier` ou `quality` separe n'est requis pour la selection de tier.
</Note>

<Warning>
Quelques IDs de modele dans le catalogue ne sont pas encore disponibles via MCP et seront rejetes en amont (variantes Sora 2 storyboard, certains paths Veo 3.1). La reponse d'erreur inclut un tableau `suggestions[]` avec jusqu'a trois alternatives proches ā€” choisissez l'une d'elles.
</Warning>

### POSTZEE_CHECK_JOB

Poll le statut d'un job asynchrone de generation d'image ou de video.

**Parametres :**

| Parametre | Type | Requis | Description |
|-----------|------|:---:|-------------|
| `jobId` | string | Oui | Job id depuis `POSTZEE_GENERATE_*` |

**Retourne (en cours) :**
```json
{ "jobId": "...", "status": "processing", "progress": 45 }
```

**Retourne (succes) :**
```json
{ "jobId": "...", "status": "success", "progress": 100, "mediaUrl": "https://cdn.postzee.app/...", "mediaId": "media-uuid" }
```

**Retourne (echec) :**
```json
{ "jobId": "...", "status": "failed", "error": "Generation failed: ..." }
```

**Retourne (introuvable) :**
```json
{ "jobId": "...", "status": "not_found" }
```

## Images et Carrousels (rendu HTML)

Le pipeline d'image et carrousel (skill v3.5+ pour carrousels, v3.7+ pour images uniques) rend du HTML fourni par l'agent en PNG via Chromium headless. Les modeles d'image IA rendent mal le texte ; le rendu HTML offre une typographie au pixel pres, des polices coherentes et zero hallucination de mots.

Cinq outils couvrent le cycle : un rendu d'image unique pour des posts editoriaux uniques avec typographie soignee, un rendu complet de carrousel pour le batch initial, remplacement chirurgical pour les corrections, append incremental pour l'authoring iteratif, plus un chemin d'upload pour les agents qui ont deja produit les bytes localement (saute completement le renderer cote serveur).

### POSTZEE_RENDER_IMAGE

Rend UN document HTML en un `Media` PNG unique. La contrepartie single-image de `POSTZEE_RENDER_CAROUSEL`, pour les posts editoriaux uniques (images chargees en texte, couvertures style magazine, hero quote cards) ou la valeur reside dans le layout + la typographie plutot que dans une sequence swipeable.

**Parametres :**

| Parametre | Type | Requis | Description |
|-----------|------|:---:|-------------|
| `html` | string | Oui | Document `<!doctype html>` complet. JavaScript est desactive au moment du rendu ā€” utilisez uniquement CSS. Embed les images comme URIs base64 `data:image/...;base64,...`. |
| `width` | number | Non | Largeur de sortie en px (256-2160). Default `1080`. |
| `height` | number | Non | Hauteur de sortie en px (256-2160). Default `1350` (4:5 ā€” optimise pour le feed Instagram). |
| `name` | string | Non | Titre optionnel affiche dans la galerie. |

**Memes limites strictes que `POSTZEE_RENDER_CAROUSEL` :** 7 MB max de HTML, 256-2160 px par dimension, 45 s de timeout.

**Retourne (succes) :**
```json
{
  "success": true,
  "mediaId": "media-uuid",
  "mediaUrl": "https://cdn.postzee.app/...",
  "width": 1080,
  "height": 1350,
  "mediaGroupId": "group-uuid",
  "message": "Image rendered successfully (1080Ć—1350). Use 'mediaUrl' when calling POSTZEE_CREATE_POST."
}
```

`mediaGroupId` est expose pour les callers avances mais l'usage typique est de passer le `mediaUrl` directement a `POSTZEE_CREATE_POST` comme un array `mediaUrls` a un seul element.

**Retourne (echec) :**
```json
{ "success": false, "error": "render_failed", "message": "..." }
```

Meme cache d'idempotency que le rendu de carrousel : des payloads identiques dans une fenetre d'1 heure retournent le `mediaId` mis en cache au lieu de re-rendre.

### POSTZEE_RENDER_CAROUSEL

Rend N slides HTML en PNG et les regroupe atomiquement en un `MediaGroup` de carrousel. **Synchrone** ā€” bloque jusqu'a ce que tous les slides soient termines.

**Parametres :**

| Parametre | Type | Requis | Description |
|-----------|------|:---:|-------------|
| `slides` | array | Oui | 1-15 objets slide dans l'ordre d'affichage. Chacun : `{ html: string, width?: number, height?: number }`. L'index du tableau devient l'ordre du slide. |
| `aspectRatio` | string | Non | Taille par defaut quand width/height sont omis : `"1:1"` (1080Ć—1080), `"4:5"` (1080Ć—1350), `"9:16"` (1080Ć—1920). Default `"1:1"`. |
| `name` | string | Non | Titre optionnel affiche dans la galerie. |

**Limites cote serveur :**
- Maximum **15 slides** par appel
- Minimum 256 / maximum **2160** px par dimension
- **7 MB** maximum de HTML par slide
- **50 MB** maximum de payload total par appel (somme de tous les slides)
- **45 s** de timeout par slide

**Garde-fous par plateforme a la publication.** Le rendu accepte jusqu'a 15 slides, mais la **publication** est limitee par le plafond natif de chaque reseau social (Instagram/Facebook ā‰¤ 10, LinkedIn ā‰¤ 20, TikTok ā‰¤ 35, Threads/Reddit ā‰¤ 20, Pinterest ā‰¤ 5, X ā‰¤ 4, Bluesky/Mastodon ā‰¤ 4, Telegram/VK ā‰¤ 10). Quand `POSTZEE_CREATE_POST` est appele avec des `mediaUrls` depassant la limite du reseau cible, Postzee rejette la publication avant d'appeler l'API de la plateforme, avec un message traduit dans la langue de l'utilisateur. Utilisez `POSTZEE_LIST_PLATFORM_SPECS` pour la table en direct.

**Modele de defaillance tolerant aux partiels.** Si un slide echoue ou si le dispatcher time out avant que tous les slides se stabilisent, le groupe est preserve avec `renderStatus: "partial"` dans `aiMetadata` (au lieu d'etre rollback). Reessayez les slides manquants via `POSTZEE_REPLACE_CAROUSEL_SLIDE` ou `POSTZEE_APPEND_CAROUSEL_SLIDE`. Les slides qui ONT rendu ne sont jamais detruits ā€” l'utilisateur conserve son travail meme en cas de defaillance partielle.

**L'ordre est structurel, pas temporel.** L'index du tableau = `orderInGroup`, assigne AVANT que tout worker tourne. Les slides peuvent rendre en parallele mais l'ordre est preserve de maniere deterministe.

**Modele de securite.** JavaScript est **desactive** dans le renderer ; toute balise `<script>` est inerte. Les requetes reseau sont interceptees et bloquees contre SSRF (RFC1918, loopback, hosts de metadata cloud).

**Retourne :**
```json
{
  "success": true,
  "mediaGroupId": "uuid",
  "totalSlides": 10,
  "aspectRatio": "4:5",
  "slides": [
    { "mediaId": "...", "url": "https://cdn.postzee.app/.../slide-0.png", "orderInGroup": 0 },
    { "mediaId": "...", "url": "https://cdn.postzee.app/.../slide-1.png", "orderInGroup": 1 }
  ],
  "mediaUrls": [
    "https://cdn.postzee.app/.../slide-0.png",
    "https://cdn.postzee.app/.../slide-1.png"
  ]
}
```

`mediaUrls` est deja dans l'ordre d'affichage ā€” passez-le directement a `POSTZEE_CREATE_POST`.

**Reponse d'echec :**
```json
{ "success": false, "error": "render_failed", "message": "Carousel rendering failed: ..." }
```

### POSTZEE_REPLACE_CAROUSEL_SLIDE

Remplace chirurgicalement UN slide d'un carrousel existant sans toucher les autres. A utiliser quand l'utilisateur dit "change le slide N" ā€” economise du temps et des credits, et preserve l'identite des autres slides (URLs et IDs intacts).

**Parametres :**

| Parametre | Type | Requis | Description |
|-----------|------|:---:|-------------|
| `mediaGroupId` | string | Oui | L'id retourne par `POSTZEE_RENDER_CAROUSEL`. |
| `orderInGroup` | number | Oui | Index base zero du slide a remplacer. Slide 4 ā†’ `3`. Doit pointer vers un slide actif (non soft-deleted). |
| `slide` | object | Oui | `{ html, width?, height? }` ā€” meme structure qu'une entree de `RENDER_CAROUSEL.slides`. |

Le slide remplace est soft-delete **apres** que le nouveau rendu est uploade avec succes ; le nouveau slide prend le meme `orderInGroup`. Si `orderInGroup === 0`, le thumbnail de couverture du carrousel est mis a jour automatiquement.

**Retourne :**
```json
{
  "success": true,
  "mediaGroupId": "uuid",
  "replacedOrderInGroup": 3,
  "newMediaId": "...",
  "newMediaUrl": "https://cdn.postzee.app/.../slide-3-{ts}.png",
  "totalSlides": 10,
  "slides": [ /* ... liste ordonnee complete ... */ ],
  "mediaUrls": [ /* ... liste ordonnee complete ... */ ],
  "message": "Slide 4 replaced. The other slides are unchanged. If you already attached the carousel to a draft post, refresh the attachments ā€” the old slide URL is now soft-deleted."
}
```

### POSTZEE_APPEND_CAROUSEL_SLIDE

Ajoute UN nouveau slide a la **fin** d'un carrousel existant. A utiliser pour l'**authoring iteratif** ā€” quand l'utilisateur veut construire le carrousel slide par slide ("montre-moi le slide 1ā€¦ maintenant le slide 2ā€¦"), ceci fait que chaque nouveau slide atterrit dans le MEME `MediaGroup` au lieu de produire N groupes de slide unique orphelins dans la galerie.

**Parametres :**

| Parametre | Type | Requis | Description |
|-----------|------|:---:|-------------|
| `mediaGroupId` | string | Oui | L'id du carrousel que vous agrandissez. Retourne par `POSTZEE_RENDER_CAROUSEL` (premier slide) ou par un appel precedent de `APPEND_CAROUSEL_SLIDE`. |
| `slide` | object | Oui | `{ html, width?, height? }` ā€” meme structure qu'une entree de `RENDER_CAROUSEL.slides`. |

Le nouveau slide est ajoute au **prochain `orderInGroup` disponible** (gap-safe ā€” si des slides intermediaires ont ete supprimes, on prend `max(orderInGroup) + 1`, jamais on ne reutilise un emplacement libre). Le thumbnail de couverture n'est pas modifie.

**Pattern d'authoring iteratif.** Le premier slide doit toujours passer par `POSTZEE_RENDER_CAROUSEL` (c'est ce qui cree le `MediaGroup`). Chaque slide suivant passe par `APPEND_CAROUSEL_SLIDE` avec le meme `mediaGroupId`. Appeler `RENDER_CAROUSEL` une seconde fois pour le meme carrousel logique produit un nouveau groupe et casse le flux iteratif ā€” ne le faites pas.

**Retourne :**
```json
{
  "success": true,
  "mediaGroupId": "uuid",
  "appendedOrderInGroup": 7,
  "newMediaId": "...",
  "newMediaUrl": "https://cdn.postzee.app/.../slide-7-{ts}.png",
  "totalSlides": 8,
  "slides": [ /* ... liste ordonnee complete incluant le nouveau ... */ ],
  "mediaUrls": [ /* ... liste ordonnee complete ... */ ]
}
```

<Warning>
Il n'y a pas de primitive d'insertion-au-milieu, de reordonnancement, ou de suppression en v1. `APPEND_CAROUSEL_SLIDE` n'ajoute qu'a la FIN. Si l'utilisateur demande d'inserer a une position specifique, d'echanger deux slides ou d'en supprimer un, recomposez tout le carrousel avec `POSTZEE_RENDER_CAROUSEL` a partir d'un nouveau scenario.
</Warning>

## Upload pre-rendu (v3.7+)

Quand l'agent tourne dans une surface avec un navigateur headless utilisable (ex. : Claude Code avec Bash + Playwright), il peut rendre le HTML en PNG localement et uploader les bytes directement ā€” sautant le renderer cote serveur de Postzee pour un round-trip plus rapide. Meme sortie `Media` / `MediaGroup` que le chemin HTML-render, donc le code en aval (publication, galerie, REST API) ne change pas.

Si l'agent n'a pas la capacite de rendu local, il bascule de maniere transparente vers `POSTZEE_RENDER_IMAGE` / `POSTZEE_RENDER_CAROUSEL`.

### POSTZEE_UPLOAD_RENDERED_IMAGE

Upload une image pre-rendue unique (PNG / JPEG / WebP) et retourne un `Media` standalone. Miroir de `POSTZEE_RENDER_IMAGE` du point de vue du code en aval ā€” meme shape de retour.

**Parametres :**

| Parametre | Type | Requis | Description |
|-----------|------|:---:|-------------|
| `imageBase64` | string | Oui | Bytes de l'image en base64. Max 10 MB raw par image. Peut inclure le prefixe `data:image/...;base64,...` ou juste le payload ā€” les deux sont acceptes. |
| `mimeType` | string | Oui | Doit correspondre aux bytes reels. Autorises : `image/png`, `image/jpeg`, `image/webp`. Le spoofing de content-type est rejete par validation magic-byte. |
| `width` | number | Oui | Largeur de sortie en px (256-2160). Doit correspondre aux dimensions reelles en pixels des bytes (validee contre le header du fichier). |
| `height` | number | Oui | Hauteur de sortie en px (256-2160). Meme validation que width. |
| `name` | string | Non | Titre optionnel affiche dans la galerie. |

**Retourne (succes) :**
```json
{
  "success": true,
  "mediaId": "media-uuid",
  "mediaUrl": "https://cdn.postzee.app/...",
  "width": 1080,
  "height": 1350,
  "message": "Image uploaded (1080Ć—1350). Use 'mediaUrl' when calling POSTZEE_CREATE_POST."
}
```

**Hardening de validation** (ces outils acceptent des bytes non fiables) :
- **Magic-byte check** contre le `mimeType` declare ā€” un fichier HTML pretendant etre `image/png` est rejete.
- **Dimension check** ā€” le `width`/`height` declare par l'agent doit correspondre aux dimensions reelles en pixels parsees du header du fichier (tolerance de Ā±2 px). Envoyer `width: 2160` avec les bytes d'une image 100Ć—100 echoue rapidement.
- **Size cap** ā€” 10 MB par image ; depasser leve une erreur claire.

Cache d'idempotency par image (1 heure) : re-uploader des bytes identiques retourne le `mediaId` mis en cache au lieu de dupliquer.

### POSTZEE_UPLOAD_RENDERED_CAROUSEL

Upload N images pre-rendues et les assemble en un `MediaGroup`. Miroir de `POSTZEE_RENDER_CAROUSEL` du point de vue du code en aval.

**Parametres :**

| Parametre | Type | Requis | Description |
|-----------|------|:---:|-------------|
| `slides` | array | Oui | 1-15 objets slide en ordre d'affichage. Chacun : `{ imageBase64, mimeType, width, height }`. Meme validation par slide que `POSTZEE_UPLOAD_RENDERED_IMAGE`. Total des bytes sur tous les slides limite a 70 MB. |
| `aspectRatio` | string | Non | `"1:1"` / `"4:5"` / `"9:16"` ā€” uniquement metadonnee, ne redimensionne pas les images. Default `"1:1"`. |
| `name` | string | Non | Titre optionnel affiche dans la galerie. |

**Limites strictes :**
- 1-15 slides par appel
- 10 MB max par slide
- 70 MB max de payload total par appel

**Retourne (succes) :**
```json
{
  "success": true,
  "mediaGroupId": "group-uuid",
  "totalSlides": 9,
  "aspectRatio": "4:5",
  "mediaUrls": [
    "https://cdn.postzee.app/.../slide-0.png",
    "https://cdn.postzee.app/.../slide-1.png",
    "..."
  ],
  "message": "Uploaded 9 slide(s). Use 'mediaUrls' (already in display order) when calling POSTZEE_CREATE_POST."
}
```

Meme semantique atomique que `POSTZEE_RENDER_CAROUSEL` : tout slide echouant a la validation abandonne tout l'upload et soft-delete le groupe partiel. Le carrousel atterrit complet ou n'atterrit pas du tout.

### POSTZEE_CREATE_POST

Cree, programme ou publie un post sur un canal de reseaux sociaux.

**Parametres :**

|-----------|------|:---:|-------------|
| `type` | string | Oui | `"now"` (publie immediatement), `"draft"` ou `"schedule"` |

**Retourne (succes) :**
```json
{ "success": true, "postId": "post-uuid", "message": "Post published successfully." }
```

**Retourne (echec) :**
```json
{ "success": false, "error": "<error_code>", "message": "...", "upgradeUrl": "https://dashboard.postzee.app/billing" }
```

**Codes d'`error` possibles :**

| Code | Signification |
|------|---------------|
| `subscription_required` | Le forfait actuel ne permet pas de publier ou la limite mensuelle de posts a ete atteinte. `upgradeUrl` est inclus. |
| `date_required` | `type: "schedule"` envoye sans `date`. |
| `invalid_date` | La string `date` n'est pas un timestamp UTC ISO valide. |
| `organization_not_found` | La cle MCP a resolu vers une organisation qui n'existe pas (rare). |

## Pattern de workflow asynchrone

La generation d'image, de video et HeyGen est asynchrone. Le workflow recommande :

1. Validez d'abord ā€” `POSTZEE_VALIDATE_GENERATION` pour capturer les erreurs de parametres et les credits insuffisants sans bruler de credits
2. Estimez le cout ā€” `POSTZEE_ESTIMATE_GENERATION_COST` si vous avez besoin du nombre exact a montrer a l'utilisateur
3. Optimisez le prompt ā€” `POSTZEE_ENHANCE_PROMPT` (gratuit)
4. Generez ā€” `POSTZEE_GENERATE_IMAGE` / `POSTZEE_GENERATE_VIDEO` / `POSTZEE_GENERATE_HEYGEN_VIDEO` ā†’ retourne `jobId`
5. Pollez ā€” `POSTZEE_CHECK_JOB` toutes les quelques secondes. Latences typiques : 10-60s pour les images, 30-180s pour les videos, jusqu'a 5min pour HeyGen
6. En cas de succes ā€” utilisez `mediaUrl` dans `POSTZEE_CREATE_POST`

## Format d'erreur standard

Tous les outils write/generate retournent :

```json
{
  "success": true | false,
  "error": "<machine_code>",
  "message": "<human readable>",
  "...": "champs specifiques a l'outil en cas de succes"
}
```

Branchez sur le `error` machine code pour la logique ; utilisez `message` pour narrer a l'utilisateur (traduit dans sa langue).
