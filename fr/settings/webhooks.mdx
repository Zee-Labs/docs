Les webhooks vous permettent de recevoir des notifications en temps réel de Postzee chaque fois qu'une publication est diffusée. Créez des intégrations et des automatisations personnalisées en connectant Postzee à vos propres systèmes.

## Comment y accéder

Ouvrez le tiroir du compte (cliquez sur votre avatar en haut à droite) → **Paramètres** → onglet **Webhooks**.

## Qu'est-ce qu'un webhook ?

Un webhook est une URL que Postzee appelle (HTTP POST) lorsque des publications sont diffusées dans votre compte. Postzee envoie une charge utile JSON contenant les détails de la publication, et votre serveur peut y réagir comme bon vous semble.

Cas d'usage courants :

- **Notifier votre équipe sur Slack/Discord** lorsqu'une publication est diffusée
- **Mettre à jour un CRM** lorsque le contenu d'un client est mis en ligne
- **Journaliser des événements** dans votre système d'analyse ou de surveillance
- **Synchroniser des données** vers une base de données personnalisée

## Comment configurer

1. Accédez à **Paramètres** → **Webhooks**
2. Cliquez sur **Ajouter un Webhook**
3. Saisissez :
   - **URL** — le point de terminaison que votre serveur expose pour recevoir les événements
   - **Nom** — une étiquette conviviale pour ce webhook
   - **Canaux** — choisissez **Tous les canaux** ou sélectionnez des canaux spécifiques à surveiller
4. Cliquez sur **Enregistrer**

Postzee commencera à envoyer les publications diffusées vers votre URL.

## Filtrage par canal

Vous pouvez limiter chaque webhook à des canaux spécifiques :

- **Tous les canaux** — recevez les événements pour chaque canal connecté
- **Canaux spécifiques** — ne se déclenche que lorsque des publications sont diffusées sur les canaux sélectionnés

Ceci est utile lorsque vous souhaitez avoir des webhooks différents pour différentes plateformes ou clients.

## Format de la charge utile

Chaque livraison de webhook est une requête HTTP POST avec un en-tête `Content-Type: application/json`. Le corps est un tableau JSON des publications récemment diffusées :

```json
[
  {
    "id": "post-uuid",
    "content": "Post text content...",
    "publishDate": "2026-04-29T15:00:00Z",
    "integration": {
      "id": "channel-uuid",
      "name": "My Instagram",
      "providerIdentifier": "instagram"
    }
  }
]
```

Les webhooks sont regroupés par lots : si plusieurs publications sont diffusées à peu près au même moment, elles peuvent être livrées ensemble dans une seule requête.
