---
title: Migración — Integrations a Channels
icon: arrows-rotate
---

La API REST pública ahora usa **channels** (canales) en vez de **integrations** (integraciones) — alineada con la UI del panel y la capa MCP. Las rutas anteriores siguen funcionando y serán retiradas el **13/11/2026**.

## Por qué este cambio

El panel de Postzee, el servidor MCP, la skill de Claude Code y el nodo de n8n ya usan la palabra **channel** para describir una cuenta social conectada. La API REST pública era la última superficie que aún usaba el término heredado **integration**. Esta migración alinea a todos los consumidores en torno a un vocabulario consistente.

## Ventana de obsolescencia

| Fecha | Evento |
|-------|--------|
| 13/05/2026 | `/public/v1/channels` lanzado. `/public/v1/integrations` marcado como obsoleto. |
| 13/11/2026 | `/public/v1/integrations` heredado es **eliminado**. Las llamadas devuelven `410 Gone`. |

Durante la ventana, las llamadas heredadas funcionan normalmente, pero devuelven:

```
Deprecation: true
Sunset: Fri, 13 Nov 2026 00:00:00 GMT
Link: </public/v1/channels>; rel="successor-version"
Warning: 299 - "Deprecated API — use /public/v1/channels"
```

## Qué cambió

### 1. Endpoint de listado

| Antes | Después |
|-------|---------|
| `GET /public/v1/integrations` | `GET /public/v1/channels` |

### 2. Nombre del campo en la respuesta

| Antes (`identifier`) | Después (`platform`) |
|---|---|
| `"identifier": "instagram"` | `"platform": "instagram"` |

Los valores en sí no cambiaron — mismos identificadores (`instagram`, `linkedin-page`, `tiktok`, `x`, etc.). Solo cambió el nombre del campo.

### 3. Formato del cuerpo de creación de post

| Antes | Después |
|-------|---------|
| `posts[].integration.id` | `posts[].channel.id` |

```diff
  {
    "type": "schedule",
    "date": "2026-05-01T14:00:00Z",
    "posts": [
      {
-       "integration": { "id": "channel-uuid" },
+       "channel": { "id": "channel-uuid" },
        "value": [ ... ]
      }
    ]
  }
```

<Warning>
Enviar **ambos** `channel` e `integration` en el mismo objeto de post devuelve `400 Bad Request`. Usa uno u otro.
</Warning>

### 4. Nuevo endpoint — obtener un canal individual

```
GET /public/v1/channels/:id
```

La API anterior no exponía esta ruta. Útil para refrescar los metadatos de un canal sin pagar el costo de un listado completo.

## Qué NO cambió

- Los valores de los identificadores de plataforma (`instagram`, `tiktok`, etc.)
- Autenticación (header `Authorization`, misma API key)
- Rate limits
- La respuesta de `GET /public/v1/posts` — sigue incluyendo el objeto heredado `integration` (con `identifier`). Cambiará en una futura versión mayor.
- Todos los demás endpoints (`/posts`, `/media`, `/credits`, `/jobs`, `/find-slot`)

## Pasos de migración

### Si llamas a la API REST directamente

1. Reemplaza `GET /public/v1/integrations` por `GET /public/v1/channels`.
2. En el parseo de la respuesta, renombra `identifier` → `platform`.
3. En el cuerpo de `POST /public/v1/posts`, renombra `integration` → `channel` dentro de cada objeto de post.

### Si usas el nodo personalizado de n8n

Actualiza `n8n-nodes-postzee` a **v2.0.0** o superior. No es necesario cambiar workflows — el nodo maneja la traducción internamente. Los workflows guardados siguen funcionando después de la actualización.

### Si usas el servidor MCP de Postzee o la skill de Claude Code

No se requiere ninguna acción. Ambos ya usan la terminología moderna `channel` / `platform`.

## Ejemplos de código

### Antes (heredado)

```bash
curl -H "Authorization: YOUR_API_KEY" \
  https://api.postzee.app/public/v1/integrations

# Respuesta: [{ "id": "...", "identifier": "instagram", ... }]

curl -X POST -H "Authorization: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "now",
    "posts": [{
      "integration": { "id": "channel-uuid" },
      "value": [{ "content": "Hola", "id": "uuid", "image": [] }]
    }]
  }' \
  https://api.postzee.app/public/v1/posts
```

### Después (moderno)

```bash
curl -H "Authorization: YOUR_API_KEY" \
  https://api.postzee.app/public/v1/channels

# Respuesta: [{ "id": "...", "platform": "instagram", ... }]

curl -X POST -H "Authorization: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "now",
    "posts": [{
      "channel": { "id": "channel-uuid" },
      "value": [{ "content": "Hola", "id": "uuid", "image": [] }]
    }]
  }' \
  https://api.postzee.app/public/v1/posts
```
