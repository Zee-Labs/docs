---
title: Migração — Integrations para Channels
icon: arrows-rotate
---

A API REST pública agora usa **channels** (canais) em vez de **integrations** (integrações) — alinhada à UI do dashboard e ao MCP. As rotas anteriores continuam funcionando e serão removidas em **13/11/2026**.

## Por que a mudança

O dashboard do Postzee, o servidor MCP, a skill do Claude Code e o nó do n8n já usam a palavra **channel** para descrever uma conta social conectada. A API REST pública era a última superfície usando o termo legado **integration**. Esta migração alinha todos os consumidores em torno de um vocabulário consistente.

## Janela de descontinuação

| Data | Evento |
|------|--------|
| 13/05/2026 | `/public/v1/channels` lançado. `/public/v1/integrations` marcado como descontinuado. |
| 13/11/2026 | `/public/v1/integrations` legado é **removido**. Chamadas retornam `410 Gone`. |

Durante a janela, chamadas ao endpoint legado funcionam normalmente, mas retornam:

```
Deprecation: true
Sunset: Fri, 13 Nov 2026 00:00:00 GMT
Link: </public/v1/channels>; rel="successor-version"
Warning: 299 - "Deprecated API — use /public/v1/channels"
```

## O que mudou

### 1. Endpoint de listagem

| Antes | Depois |
|-------|--------|
| `GET /public/v1/integrations` | `GET /public/v1/channels` |

### 2. Nome do campo na resposta

| Antes (`identifier`) | Depois (`platform`) |
|---|---|
| `"identifier": "instagram"` | `"platform": "instagram"` |

Os valores em si não mudaram — mesmos identificadores (`instagram`, `linkedin-page`, `tiktok`, `x`, etc.). Apenas o nome do campo foi renomeado.

### 3. Formato do corpo de criação de post

| Antes | Depois |
|-------|--------|
| `posts[].integration.id` | `posts[].channel.id` |

```diff
  {
    "type": "schedule",
    "date": "2026-05-01T14:00:00Z",
    "posts": [
      {
-       "integration": { "id": "channel-uuid" },
+       "channel": { "id": "channel-uuid" },
        "value": [ ... ]
      }
    ]
  }
```

<Warning>
Enviar **ambos** `channel` e `integration` no mesmo objeto de post retorna `400 Bad Request`. Use um ou outro.
</Warning>

### 4. Novo endpoint — obter um canal específico

```
GET /public/v1/channels/:id
```

A API anterior não expunha esta rota. Útil para atualizar os metadados de um canal sem pagar o custo de uma listagem completa.

## O que NÃO mudou

- Os valores dos identificadores de plataforma (`instagram`, `tiktok`, etc.)
- Autenticação (header `Authorization`, mesma API key)
- Rate limits
- A resposta de `GET /public/v1/posts` — continua embutindo o objeto legado `integration` (com `identifier`). Mudará em uma futura versão maior.
- Todos os outros endpoints (`/posts`, `/media`, `/credits`, `/jobs`, `/find-slot`)

## Passos de migração

### Se você chama a API REST diretamente

1. Substitua `GET /public/v1/integrations` por `GET /public/v1/channels`.
2. No parsing da resposta, renomeie `identifier` → `platform`.
3. No corpo de `POST /public/v1/posts`, renomeie `integration` → `channel` dentro de cada objeto de post.

### Se você usa o nó customizado do n8n

Atualize `n8n-nodes-postzee` para **v2.0.0** ou superior. Não é preciso mudar workflows — o nó faz a tradução internamente. Workflows salvos continuam funcionando após o upgrade.

### Se você usa o servidor MCP do Postzee ou a skill do Claude Code

Nenhuma ação necessária. Ambos já usam a terminologia moderna `channel` / `platform`.

## Exemplos de código

### Antes (legado)

```bash
curl -H "Authorization: YOUR_API_KEY" \
  https://api.postzee.app/public/v1/integrations

# Resposta: [{ "id": "...", "identifier": "instagram", ... }]

curl -X POST -H "Authorization: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "now",
    "posts": [{
      "integration": { "id": "channel-uuid" },
      "value": [{ "content": "Olá", "id": "uuid", "image": [] }]
    }]
  }' \
  https://api.postzee.app/public/v1/posts
```

### Depois (moderno)

```bash
curl -H "Authorization: YOUR_API_KEY" \
  https://api.postzee.app/public/v1/channels

# Resposta: [{ "id": "...", "platform": "instagram", ... }]

curl -X POST -H "Authorization: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "now",
    "posts": [{
      "channel": { "id": "channel-uuid" },
      "value": [{ "content": "Olá", "id": "uuid", "image": [] }]
    }]
  }' \
  https://api.postzee.app/public/v1/posts
```
