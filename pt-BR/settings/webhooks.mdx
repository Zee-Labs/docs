Webhooks permitem que você receba notificações em tempo real do Postzee sempre que um post for publicado. Crie integrações e automações personalizadas conectando o Postzee aos seus próprios sistemas.

## Como acessar

Abra o drawer da conta (clique no seu avatar no canto superior direito) → **Configurações** → aba **Webhooks**.

## O que é um webhook?

Um webhook é uma URL que o Postzee chama (HTTP POST) quando posts são publicados na sua conta. O Postzee envia um payload JSON com os detalhes do post, e o seu servidor pode reagir a ele da forma que desejar.

Casos de uso comuns:

- **Notificar seu time no Slack/Discord** quando um post for publicado
- **Atualizar um CRM** quando o conteúdo de um cliente entrar no ar
- **Registrar eventos** no seu sistema de analytics ou monitoramento
- **Sincronizar dados** com um banco de dados personalizado

## Como configurar

1. Acesse **Configurações** → **Webhooks**
2. Clique em **Adicionar Webhook**
3. Informe:
   - **URL** — o endpoint que seu servidor expõe para receber os eventos
   - **Nome** — um rótulo amigável para este webhook
   - **Canais** — escolha **Todos os canais** ou selecione canais específicos para monitorar
4. Clique em **Salvar**

O Postzee começará a enviar os posts publicados para a sua URL.

## Filtragem por canal

Você pode restringir cada webhook a canais específicos:

- **Todos os canais** — receba eventos de todos os canais conectados
- **Canais específicos** — dispare apenas quando posts forem publicados nos canais selecionados

Isso é útil quando você quer webhooks diferentes para plataformas ou clientes diferentes.

## Formato do payload

Toda entrega de webhook é um HTTP POST com o cabeçalho `Content-Type: application/json`. O corpo é um array JSON com os posts publicados recentemente:

```json
[
  {
    "id": "post-uuid",
    "content": "Post text content...",
    "publishDate": "2026-04-29T15:00:00Z",
    "integration": {
      "id": "channel-uuid",
      "name": "My Instagram",
      "providerIdentifier": "instagram"
    }
  }
]
```

Webhooks são entregues em lote: se vários posts forem publicados aproximadamente ao mesmo tempo, eles podem ser enviados juntos em uma única requisição.
