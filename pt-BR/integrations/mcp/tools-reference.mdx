Referência completa para todas as **22 ferramentas** expostas pelo servidor MCP do Postzee. Essas ferramentas estão disponíveis para qualquer agente IA compatível com MCP (Claude Code, OpenClaw, Hermes Agent ou clientes MCP diretos).

<Note>
Todas as ferramentas aceitam a chave de API embutida na sua URL MCP. Elas são escopadas a uma única organização. **Custos de geração de IA são sempre retornados em créditos, nunca em dólares.** ($1 USD = 1.000 créditos.)
</Note>

## Referência rápida

| Categoria | Ferramentas |
|-----------|-------------|

## Modelos e Specs

### POSTZEE_LIST_MODELS_DETAILED

Matriz de capacidades para todos os modelos de imagem e vídeo disponíveis — durações, resoluções, capacidades de áudio, parâmetros customizados aceitos, e tier de custo relativo.

**Parâmetros:**

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|:---:|-----------|
| `type` | string | Não | `"image"`, `"video"`, ou `"all"` (default) |
| `includeHeygen` | boolean | Não | Inclui a entrada sintética HeyGen. Default: `true` |
| `family` | string | Não | Filtra por uma única família (ex.: `"ideogram-v3"`, `"sora-2"`, `"veo3.1"`) |
| `agenticDefault` | boolean | Não | Quando `true`, retorna apenas as entradas marcadas como recomendação para automação |

**Retorna (por modelo):**
```json
{
  "modelId": "sora-2-t2v-pro-1080p",
  "displayName": "Sora 2 Pro 1080p (Text-to-Video)",
  "type": "video",
  "family": "sora-2-pro",
  "tierWithinFamily": "premium",
  "audio": "speech-lipsync",
  "supportsImg2Img": false,
  "supportsImg2Vid": false,
  "supportsVector": false,
  "durations": [10, 15],
  "resolutions": ["1080p"],
  "aspectRatios": ["16:9", "9:16"],
  "recommendedAspectsByPlatform": {
    "instagram_reels": "9:16",
    "tiktok_feed": "9:16",
    "youtube_feed": "16:9"
  },
  "customParamsAccepted": {
    "duration": { "type": "enum", "enum": [10, 15], "description": "..." }
  },
  "costTier": "premium",
  "popular": false,
  "agenticDefault": false,
  "agenticDefaultFree": false,
  "bestFor": ["highest cinematic quality", "1080p Pro", "final publication"],
  "notRecommendedFor": ["high-volume automation"]
}
```

`costTier` é um de `very-low`, `low`, `mid`, `high`, `premium` — o custo absoluto em créditos não é exposto intencionalmente; use `POSTZEE_ESTIMATE_GENERATION_COST` em vez disso.

A entrada sintética `heygen-avatar-video` tem `costTier: "external"` e um `billingNote` indicando que o HeyGen cobra na conta HeyGen do usuário, não em créditos Postzee.

**Parâmetros:**

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|:---:|-----------|
| `type` | string | Sim | `"image"` ou `"video"` |

### POSTZEE_VALIDATE_GENERATION

Validação pré-execução — captura erros de parâmetros, créditos insuficientes, problemas de storage e bloqueios de plano **antes** de queimar créditos.

**Parâmetros:**

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|:---:|-----------|
| `type` | string | Sim | `"image"` ou `"video"` |
| `model` | string | Sim | ID do modelo |
| `duration` | number | Não | Segundos |
| `aspectRatio` | string | Não | ex.: `"4:5"` |
| `resolution` | string | Não | Resolução de saída |
| `quality` | string | Não | Sora Pro `"standard"` / `"high"` |
| `generateAudio` | boolean | Não | |
| `voiceControl` | boolean | Não | |
| `imageUrl` | string | Não | Obrigatório para modelos image-to-video |
| `slideCount` | number | Não | Multiplicador de batch (ex.: N variações do mesmo prompt). **Não use para carrosséis** — esses passam por `POSTZEE_RENDER_CAROUSEL` com modelo de custo próprio. |

**Retorna:**
```json
{
  "valid": true,
  "errors": [],
  "warnings": ["Storage at 85% — approaching limit."],
  "estimatedCreditsPerItem": 160,
  "totalEstimatedCredits": 1600,
  "slideCount": 10,
  "currentBalance": 4500,
  "willExceedBalance": false,
  "shortfall": 0
}
```

Quando `valid: false`, o array `errors[]` contém problemas legíveis — ex.: duração não está no conjunto permitido pelo modelo, falta `imageUrl` para modelo i2v, storage em 100%, ou modelo registrado mas não disponível ainda via MCP.

## Canais e Créditos

### POSTZEE_LIST_CHANNELS

Lista canais de redes sociais conectados com flags de ação explícitas.

**Parâmetros:** Nenhum

**Retorna:**
```json
[
  {
    "id": "channel-uuid",
    "name": "My Instagram",
    "platform": "instagram",
    "picture": "https://...",
    "profile": "https://...",
    "disabled": false,
    "requiresReauth": false,
    "canPostNow": true,
    "actionRequired": "none",
    "actionUrl": null,
    "actionMessage": null,
    "status": "active",
    "statusMessage": null
  }
]
```

`actionRequired` é um de `"none"` (pronto para postar), `"reconnect"` (token expirado — usuário reconecta), ou `"reenable"` (admin/billing — usuário reativa, **não** o mesmo que reconectar).

`statusMessage` é uma explicação legível quando o canal requer ação (ex: `"Token do canal expirado. Reconecte para continuar postando."`). É `null` quando `actionRequired` é `"none"`.

## Memória de Mídia

As duas ferramentas abaixo permitem que o agente recupere e reutilize ativos de mídia — gerados por IA ou enviados pelo usuário — entre turnos e entre sessões, sem precisar pedir URLs ao usuário novamente.

### POSTZEE_LIST_MEDIA

Lista de mídia recente, escopada à organização. Retorna os itens mais novos primeiro.

**Parâmetros:**

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|:---:|-------------|
| `type` | string | Não | `"image"`, `"video"` ou `"all"` (padrão) |
| `source` | string | Não | `"generated"` (criado por IA), `"uploaded"` (importado via `POSTZEE_UPLOAD_MEDIA`) ou `"all"` (padrão) |
| `search` | string | Não | Busca case-insensitive no prompt original (mídia IA) e no nome do arquivo |
| `since` | string | Não | Data ISO — retorna apenas mídia criada após esse instante |
| `limit` | number | Não | Padrão 20, máximo 100 |

**Retorna (por item):**
```json
{
  "mediaId": "uuid",
  "url": "https://cdn.postzee.app/abc.jpg",
  "name": "abc.jpg",
  "type": "image",
  "source": "generated",
  "prompt": "cachorro na praia ao pôr do sol",
  "modelUsed": "ideogram-v3-turbo",
  "createdAt": "2026-05-07T14:23:00Z",
  "sizeBytes": 245678,
  "dimensions": { "width": 1024, "height": 1280 }
}
```

**Retorna (falha):**
```json
{ "error": "list_media_failed", "message": "..." }
```

<Note>
Mídias com soft-delete são excluídas automaticamente. Mídia em uso como foto de perfil também é excluída.
</Note>

### POSTZEE_UPLOAD_MEDIA

Importa uma URL pública para o storage do Postzee. Retorna um `mediaId` estável e URL na CDN para reuso em chamadas `GENERATE_*`, `CREATE_POST` e `LIST_MEDIA`.

**Parâmetros:**

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|:---:|-------------|
| `url` | string | Sim | URL pública `http://` ou `https://`. Deve ser acessível sem autenticação. |
| `description` | string | Não | Rótulo curto guardado junto à mídia (ajuda o agente a manter seu manifest) |

**Retorna (sucesso):**
```json
{
  "success": true,
  "mediaId": "uuid",
  "url": "https://cdn.postzee.app/uploads/xyz.jpg",
  "type": "image",
  "sizeBytes": 524288,
  "dimensions": { "width": 1080, "height": 1920 }
}
```

**Retorna (falha):**
```json
{
  "success": false,
  "error": "blocked_url | unsupported_type | size_limit_exceeded | fetch_failed | storage_full | upload_failed | save_failed",
  "message": "..."
}
```

**Ordem de validação (cada passo rejeita antes do próximo):**

1. URL é parseável e usa `http`/`https`
2. Hostname **não** é localhost / IP privado / endpoint de cloud-metadata (defesa SSRF)
3. Sondagem HEAD bem-sucedida (ou GET ranged como fallback)
4. Content-Type bate com a allowlist configurada (padrão: `image/jpeg`, `image/png`, `image/webp`, `image/gif` / `video/mp4`, `video/quicktime`, `video/webm`)
5. Tamanho dentro do cap por tipo (padrão: 25 MB imagem, 500 MB vídeo)
6. Quota de storage da organização tem espaço

**Limites padrão (configuráveis via SystemConfig):**

| Tipo | Tamanho máximo | Formatos aceitos |
|------|----------------|------------------|
| Imagem | 25 MB | JPG, PNG, WebP, GIF |
| Vídeo | 500 MB | MP4, MOV, WebM |

Limites e formatos aceitos podem ser ajustados em runtime alterando as chaves `UPLOAD_*` no `SystemConfig`. Tanto esta ferramenta MCP quanto os endpoints REST de upload compartilham a mesma fonte de verdade — altera uma vez, vale para todos em ~5 minutos (TTL do cache).

<Warning>
Por privacidade e segurança, a ferramenta nunca aceita headers de autenticação ou tokens. URLs que requerem Bot Token (ex.: URLs `getFile` do Telegram) precisam ser re-hospedadas pelo cliente de chat em uma URL pública primeiro. Para uploads em base64 / arquivos diretos, use o endpoint REST existente `/media/upload-server` — a ferramenta MCP só lida com importações via URL.
</Warning>

<Note>
Após importação bem-sucedida, a mídia aparece em `POSTZEE_LIST_MEDIA` com `source: "uploaded"`.
</Note>

## Otimização de Prompt

### POSTZEE_ENHANCE_PROMPT

Otimiza o prompt do usuário para melhores resultados de geração de IA. **Grátis** — sem custo de créditos.

**Parâmetros:**

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|:---:|-----------|
| `prompt` | string | Sim | O prompt do usuário a ser otimizado |
| `mediaType` | string | Sim | `"image"` ou `"video"` |
| `model` | string | Não | ID do modelo alvo (para otimização específica do modelo) |
| `language` | string | Não | Código do idioma de saída (ex: `"pt"`, `"es"`, `"fr"`, `"de"`). Se omitido, usa o idioma do proprietário da conta ou padrão inglês. |

**Retorna:**
```json
{ "success": true, "enhancedPrompt": "A young woman in a tailored trench coat walks through..." }
```

## Geração

### POSTZEE_GENERATE_IMAGE

Gera uma imagem IA a partir de um prompt. **Operação assíncrona** — retorna um job id para fazer poll com `POSTZEE_CHECK_JOB`.

**Parâmetros:**

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|:---:|-----------|
| `prompt` | string | Sim | Descrição detalhada da imagem |
| `model` | string | Sim | ID do modelo de `POSTZEE_LIST_MODELS_DETAILED` (type `"image"`) |
| `aspectRatio` | string | Não | ex.: `"1:1"`, `"4:5"`, `"9:16"`, `"16:9"`. Default: `"1:1"` |
| `imageUrls` | string[] | Não | URLs de imagens de referência para image-to-image |
| `quality` | string | Não | GPT Image 2: `"low"`, `"medium"` ou `"high"`. Default: `"high"` |
| `style` | string | Não | Recraft V3/V4: `"realistic_image"`, `"digital_illustration"`, `"vector_illustration"` |
| `renderingSpeed` | string | Não | Tier do Ideogram V3: `"turbo"`, `"balanced"` ou `"quality"` |
| `negativePrompt` | string | Não | Coisas a excluir da imagem |
| `seed` | number | Não | Seed determinística opcional |

**Retorna (sucesso):**
```json
{ "success": true, "jobId": "job-uuid", "status": "processing", "message": "Image generation started..." }
```

**Retorna (falha):**
```json
{ "success": false, "error": "unsupported_model | unknown_model | wrong_type | organization_not_found | generation_failed", "message": "...", "suggestions": [{ "modelId": "...", "reason": "..." }] }
```

---

### POSTZEE_GENERATE_VIDEO

Gera um vídeo IA a partir de um prompt ou imagem de referência. **Operação assíncrona.**

**Parâmetros:**

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|:---:|-----------|
| `prompt` | string | Sim | Descrição detalhada do vídeo |
| `model` | string | Sim | ID do modelo de `POSTZEE_LIST_MODELS_DETAILED` (type `"video"`) |
| `duration` | number | Não | Segundos. Valores permitidos variam por modelo — consulte `model.durations` |
| `aspectRatio` | string | Não | `"16:9"` ou `"9:16"`. Default: `"16:9"` |
| `imageUrl` | string | Não | URL de imagem de referência para modelos image-to-video |
| `endImageUrl` | string | Não | URL do frame final para interpolação first-last-frame (ex.: `wan-flf2v`) |
| `resolution` | string | Não | `"480p"`, `"540p"`, `"720p"`, `"1080p"`, `"4k"` (depende do modelo) |
| `quality` | string | Não | Apenas Sora 2 Pro: `"standard"` (720p) ou `"high"` (1080p) |
| `generateAudio` | boolean | Não | Gera áudio ambiente (Veo 3.1, Pixverse, Wan 2.5). Afeta créditos. |
| `voiceControl` | boolean | Não | Controle de voz (Kling 2.6/3.0 Pro). Afeta créditos. |
| `negativePrompt` | string | Não | Coisas a evitar no vídeo |
| `seed` | number | Não | Seed determinística opcional |

**Retorna:** mesma estrutura de `POSTZEE_GENERATE_IMAGE`.

<Note>
Os IDs específicos de tier retornados por `POSTZEE_LIST_MODELS_DETAILED` (ex.: `ideogram-v3-turbo`, `gpt-image-2-high`, `sora-2-t2v-pro-1080p`, `recraft-v4-vector`) podem ser passados diretamente em `model`. O MCP traduz para o payload correto do backend automaticamente — não é necessário um parâmetro `tier` ou `quality` separado para a seleção de tier.
</Note>

<Warning>
Alguns IDs de modelo no catálogo ainda não estão disponíveis via MCP e serão rejeitados de antemão (variantes Sora 2 storyboard, alguns paths Veo 3.1). A resposta de erro inclui um array `suggestions[]` com até três alternativas próximas — escolha uma delas.
</Warning>

### POSTZEE_CHECK_JOB

Faz poll do status de um job assíncrono de geração de imagem ou vídeo.

**Parâmetros:**

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|:---:|-----------|
| `jobId` | string | Sim | Job id de `POSTZEE_GENERATE_*` |

**Retorna (processando):**
```json
{ "jobId": "...", "status": "processing", "progress": 45 }
```

**Retorna (sucesso):**
```json
{ "jobId": "...", "status": "success", "progress": 100, "mediaUrl": "https://cdn.postzee.app/...", "mediaId": "media-uuid" }
```

**Retorna (falha):**
```json
{ "jobId": "...", "status": "failed", "error": "Generation failed: ..." }
```

**Retorna (não encontrado):**
```json
{ "jobId": "...", "status": "not_found" }
```

## Carrosséis

O pipeline de carrossel (skill v3.5+) renderiza HTML fornecido pelo agente para PNG via Chromium headless. Modelos de imagem IA renderizam texto mal; o render via HTML entrega tipografia pixel-perfect, fontes consistentes e zero alucinação de palavras. Três tools cobrem o ciclo: um render completo para o batch inicial, uma substituição cirúrgica para correções, e um append incremental para autoria iterativa (construir slide por slide e ver cada um cair no mesmo carrossel crescente).

### POSTZEE_RENDER_CAROUSEL

Renderiza N slides HTML para PNG e os agrupa atomicamente como um `MediaGroup` de carrossel. **Síncrona** — bloqueia até todos os slides finalizarem.

**Parâmetros:**

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|:---:|-------------|
| `slides` | array | Sim | 1-15 objetos slide em ordem de exibição. Cada um: `{ html: string, width?: number, height?: number }`. O índice do array vira a ordem do slide. |
| `aspectRatio` | string | Não | Tamanho default quando width/height são omitidos: `"1:1"` (1080×1080), `"4:5"` (1080×1350), `"9:16"` (1080×1920). Default `"1:1"`. |
| `name` | string | Não | Título opcional exibido na galeria. |

**Limites server-side:**
- Máximo **15 slides** por chamada
- Mínimo 256 / máximo **2160** px por dimensão
- 250 KB de HTML por slide
- 30 s de timeout por slide

**Modelo de falha tolerante a parcial.** Se um slide falhar ou o dispatcher timeout antes de todos os slides assentarem, o grupo é preservado com `renderStatus: "partial"` no `aiMetadata` (em vez de ser revertido). Refaça os slides faltantes via `POSTZEE_REPLACE_CAROUSEL_SLIDE` ou `POSTZEE_APPEND_CAROUSEL_SLIDE`. Slides que renderizaram nunca são destruídos — o usuário mantém o trabalho mesmo em falha parcial.

**Ordem é estrutural, não temporal.** O índice do array = `orderInGroup`, atribuído ANTES de qualquer worker rodar. Slides podem renderizar em paralelo, mas a ordem é preservada deterministicamente.

**Modelo de segurança.** JavaScript é **desabilitado** no renderer; tags `<script>` são inertes. Requisições de rede são interceptadas e bloqueiam SSRF (RFC1918, loopback, hosts de cloud-metadata).

**Retorna:**
```json
{
  "success": true,
  "mediaGroupId": "uuid",
  "totalSlides": 10,
  "aspectRatio": "4:5",
  "slides": [
    { "mediaId": "...", "url": "https://cdn.postzee.app/.../slide-0.png", "orderInGroup": 0 },
    { "mediaId": "...", "url": "https://cdn.postzee.app/.../slide-1.png", "orderInGroup": 1 }
  ],
  "mediaUrls": [
    "https://cdn.postzee.app/.../slide-0.png",
    "https://cdn.postzee.app/.../slide-1.png"
  ]
}
```

`mediaUrls` já vem em ordem de exibição — passe direto para `POSTZEE_CREATE_POST`.

**Resposta de falha:**
```json
{ "success": false, "error": "render_failed", "message": "Carousel rendering failed: ..." }
```

### POSTZEE_REPLACE_CAROUSEL_SLIDE

Substitui cirurgicamente UM slide de um carrossel existente sem tocar nos outros. Use quando o usuário pede "muda o slide N" — economiza tempo e créditos, e preserva a identidade dos demais slides (URLs e IDs intactos).

**Parâmetros:**

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|:---:|-------------|
| `mediaGroupId` | string | Sim | O id retornado por `POSTZEE_RENDER_CAROUSEL`. |
| `orderInGroup` | number | Sim | Índice base-zero do slide a substituir. Slide 4 → `3`. Deve apontar pra slide ativo (não soft-deleted). |
| `slide` | object | Sim | `{ html, width?, height? }` — mesma estrutura de uma entrada de `RENDER_CAROUSEL.slides`. |

O slide substituído é soft-deleted **depois** que o novo render é uploadado com sucesso; o novo slide assume o mesmo `orderInGroup`. Se `orderInGroup === 0`, o thumbnail de capa do carrossel é atualizado automaticamente.

**Retorna:**
```json
{
  "success": true,
  "mediaGroupId": "uuid",
  "replacedOrderInGroup": 3,
  "newMediaId": "...",
  "newMediaUrl": "https://cdn.postzee.app/.../slide-3-{ts}.png",
  "totalSlides": 10,
  "slides": [ /* ... lista ordenada completa ... */ ],
  "mediaUrls": [ /* ... lista ordenada completa ... */ ],
  "message": "Slide 4 replaced. The other slides are unchanged. If you already attached the carousel to a draft post, refresh the attachments — the old slide URL is now soft-deleted."
}
```

### POSTZEE_APPEND_CAROUSEL_SLIDE

Adiciona UM novo slide ao **final** de um carrossel existente. Use para **autoria iterativa** — quando o usuário quer construir o carrossel slide a slide ("mostra o slide 1… agora o slide 2…"), isso mantém cada novo slide caindo no MESMO `MediaGroup` em vez de produzir N grupos de slide único órfãos na galeria.

**Parâmetros:**

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|:---:|-------------|
| `mediaGroupId` | string | Sim | O id do carrossel que você está expandindo. Retornado por `POSTZEE_RENDER_CAROUSEL` (primeiro slide) ou por uma chamada anterior de `APPEND_CAROUSEL_SLIDE`. |
| `slide` | object | Sim | `{ html, width?, height? }` — mesma estrutura de uma entrada de `RENDER_CAROUSEL.slides`. |

O novo slide é adicionado no **próximo `orderInGroup` disponível** (gap-safe — se slides intermediários foram deletados, isso pega `max(orderInGroup) + 1`, nunca reaproveita slot livre). Thumbnail de capa não é alterado.

**Padrão de autoria iterativa.** Primeiro slide ainda deve ir por `POSTZEE_RENDER_CAROUSEL` (é o que cria o `MediaGroup`). Cada slide subsequente vai por `APPEND_CAROUSEL_SLIDE` com o mesmo `mediaGroupId`. Chamar `RENDER_CAROUSEL` uma segunda vez para o mesmo carrossel lógico produz um novo grupo e quebra o fluxo iterativo — não faça.

**Retorna:**
```json
{
  "success": true,
  "mediaGroupId": "uuid",
  "appendedOrderInGroup": 7,
  "newMediaId": "...",
  "newMediaUrl": "https://cdn.postzee.app/.../slide-7-{ts}.png",
  "totalSlides": 8,
  "slides": [ /* ... lista ordenada completa incluindo o novo ... */ ],
  "mediaUrls": [ /* ... lista ordenada completa ... */ ]
}
```

<Warning>
Não há primitiva de inserir-no-meio, reordenar ou remover na v1. `APPEND_CAROUSEL_SLIDE` só adiciona ao FINAL. Se o usuário pedir pra inserir em posição específica, trocar dois slides ou remover um, refaça o carrossel inteiro com `POSTZEE_RENDER_CAROUSEL` a partir de um novo roteiro.
</Warning>

## Postagem

### POSTZEE_CREATE_POST

Cria, agenda ou publica um post em um canal de rede social.

**Parâmetros:**

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|:---:|-----------|
| `type` | string | Sim | `"now"` (publica imediatamente), `"draft"` ou `"schedule"` |
| `date` | string | Apenas schedule | UTC datetime (ex.: `"2026-04-24T15:00:00Z"`) — **obrigatório** quando `type` é `"schedule"` |
| `channelId` | string | Sim | ID do canal de `POSTZEE_LIST_CHANNELS` |
| `text` | string | Sim | Conteúdo do post |
| `mediaUrls` | string[] | Não | URLs das mídias (em ordem de exibição — ordem do array = ordem dos slides em carrosséis) |

**Retorna (sucesso):**
```json
{ "success": true, "postId": "post-uuid", "message": "Post published successfully." }
```

**Retorna (falha):**
```json
{ "success": false, "error": "<error_code>", "message": "...", "upgradeUrl": "https://dashboard.postzee.app/billing" }
```

**Códigos de `error` possíveis:**

| Código | Significado |
|--------|-------------|
| `subscription_required` | O plano atual não permite publicar ou o limite mensal de posts foi atingido. `upgradeUrl` é incluído. |
| `date_required` | `type: "schedule"` foi enviado sem `date`. |
| `invalid_date` | A string `date` não é um timestamp UTC ISO válido. |
| `organization_not_found` | A chave MCP resolveu para uma organização que não existe (raro). |
| `post_failed` | Erro de provider/rede durante a publicação. Tente uma vez de novo; se persistir, surface a `message`. |

## Padrão de workflow assíncrono

Geração de imagem, vídeo e HeyGen são assíncronas. O workflow recomendado:

1. Valide primeiro — `POSTZEE_VALIDATE_GENERATION` para capturar erros de parâmetros e créditos insuficientes sem queimar créditos
2. Estime o custo — `POSTZEE_ESTIMATE_GENERATION_COST` se você precisar do número exato para mostrar ao usuário
3. Otimize o prompt — `POSTZEE_ENHANCE_PROMPT` (grátis)
4. Gere — `POSTZEE_GENERATE_IMAGE` / `POSTZEE_GENERATE_VIDEO` / `POSTZEE_GENERATE_HEYGEN_VIDEO` → retorna `jobId`
5. Faça poll — `POSTZEE_CHECK_JOB` a cada poucos segundos. Latências típicas: 10-60s para imagens, 30-180s para vídeos, até 5min para HeyGen
6. Em caso de sucesso — use `mediaUrl` em `POSTZEE_CREATE_POST`

## Formato de erro padrão

Todas as ferramentas write/generate retornam:

```json
{
  "success": true | false,
  "error": "<machine_code>",
  "message": "<human readable>",
  "...": "campos específicos da ferramenta em sucesso"
}
```

Faça branch no `error` machine code para lógica; use `message` para narrar ao usuário (traduzido para o idioma dele).
