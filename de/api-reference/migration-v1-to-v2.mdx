---
title: Migration — Integrations zu Channels
icon: arrows-rotate
---

Die oeffentliche REST-API verwendet nun **channels** (Kanaele) statt **integrations** (Integrationen) — passend zur Dashboard-UI und zur MCP-Schicht. Die vorherigen Routen funktionieren weiterhin und werden am **13.11.2026** entfernt.

## Warum diese Aenderung

Das Postzee-Dashboard, der MCP-Server, die Claude-Code-Skill und der n8n-Node verwenden bereits das Wort **channel**, um ein verbundenes Social-Konto zu beschreiben. Die oeffentliche REST-API war die letzte Schnittstelle, die noch den Legacy-Begriff **integration** verwendete. Diese Migration vereinheitlicht alle Konsumenten auf ein konsistentes Vokabular.

## Deprecation-Fenster

| Datum | Ereignis |
|-------|----------|
| 13.05.2026 | `/public/v1/channels` veroeffentlicht. `/public/v1/integrations` als veraltet markiert. |
| 13.11.2026 | Legacy `/public/v1/integrations` wird **entfernt**. Aufrufe liefern `410 Gone`. |

Waehrend des Fensters funktionieren Legacy-Aufrufe normal, liefern aber:

```
Deprecation: true
Sunset: Fri, 13 Nov 2026 00:00:00 GMT
Link: </public/v1/channels>; rel="successor-version"
Warning: 299 - "Deprecated API — use /public/v1/channels"
```

## Was sich geaendert hat

### 1. List-Endpoint

| Vorher | Nachher |
|--------|---------|
| `GET /public/v1/integrations` | `GET /public/v1/channels` |

### 2. Feldname in der Antwort

| Vorher (`identifier`) | Nachher (`platform`) |
|---|---|
| `"identifier": "instagram"` | `"platform": "instagram"` |

Die Werte selbst haben sich nicht geaendert — gleiche Identifier (`instagram`, `linkedin-page`, `tiktok`, `x`, etc.). Nur der Feldname wurde umbenannt.

### 3. Format des Post-Erstellungs-Bodys

| Vorher | Nachher |
|--------|---------|
| `posts[].integration.id` | `posts[].channel.id` |

```diff
  {
    "type": "schedule",
    "date": "2026-05-01T14:00:00Z",
    "posts": [
      {
-       "integration": { "id": "channel-uuid" },
+       "channel": { "id": "channel-uuid" },
        "value": [ ... ]
      }
    ]
  }
```

<Warning>
Das Senden **beider** Felder `channel` und `integration` im selben Post-Objekt liefert `400 Bad Request`. Verwenden Sie eines oder das andere.
</Warning>

### 4. Neuer Endpoint — einzelnen Kanal abrufen

```
GET /public/v1/channels/:id
```

Die vorherige API hat diese Route nicht angeboten. Nuetzlich, um die Metadaten eines Kanals zu aktualisieren, ohne den Aufwand einer vollstaendigen Liste zu zahlen.

## Was sich NICHT geaendert hat

- Die Werte der Plattform-Identifier (`instagram`, `tiktok`, etc.)
- Authentifizierung (`Authorization`-Header, gleicher API-Key)
- Rate Limits
- Die Antwort von `GET /public/v1/posts` — enthaelt weiterhin das Legacy-Objekt `integration` (mit `identifier`). Dies wird sich in einer zukuenftigen Major-Version aendern.
- Alle anderen Endpoints (`/posts`, `/media`, `/credits`, `/jobs`, `/find-slot`)

## Migrations-Schritte

### Wenn Sie die REST-API direkt aufrufen

1. Ersetzen Sie `GET /public/v1/integrations` durch `GET /public/v1/channels`.
2. Im Antwort-Parsing benennen Sie `identifier` → `platform` um.
3. Im Body von `POST /public/v1/posts` benennen Sie `integration` → `channel` in jedem Post-Objekt um.

### Wenn Sie den n8n-Custom-Node verwenden

Aktualisieren Sie `n8n-nodes-postzee` auf **v2.0.0** oder hoeher. Keine Workflow-Aenderungen erforderlich — der Node behandelt die Uebersetzung intern. Gespeicherte Workflows funktionieren nach dem Upgrade weiter.

### Wenn Sie den Postzee MCP-Server oder die Claude-Code-Skill verwenden

Keine Aktion erforderlich. Beide verwenden bereits die moderne Terminologie `channel` / `platform`.

## Code-Beispiele

### Vorher (Legacy)

```bash
curl -H "Authorization: YOUR_API_KEY" \
  https://api.postzee.app/public/v1/integrations

# Antwort: [{ "id": "...", "identifier": "instagram", ... }]

curl -X POST -H "Authorization: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "now",
    "posts": [{
      "integration": { "id": "channel-uuid" },
      "value": [{ "content": "Hallo", "id": "uuid", "image": [] }]
    }]
  }' \
  https://api.postzee.app/public/v1/posts
```

### Nachher (modern)

```bash
curl -H "Authorization: YOUR_API_KEY" \
  https://api.postzee.app/public/v1/channels

# Antwort: [{ "id": "...", "platform": "instagram", ... }]

curl -X POST -H "Authorization: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "now",
    "posts": [{
      "channel": { "id": "channel-uuid" },
      "value": [{ "content": "Hallo", "id": "uuid", "image": [] }]
    }]
  }' \
  https://api.postzee.app/public/v1/posts
```
