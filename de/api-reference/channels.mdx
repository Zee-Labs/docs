Rufen Sie Ihre verbundenen Social-Media-Kanaele ab.

## Kanaele auflisten

```
GET /public/v1/channels
```

Gibt alle verbundenen Social-Media-Kanaele Ihres Kontos zurueck.

### Anfrage

```bash
curl -H "Authorization: YOUR_API_KEY" \
  https://api.postzee.app/public/v1/channels
```

### Antwort

```json
[
  {
    "id": "abc123-def456",
    "name": "My Instagram",
    "platform": "instagram",
    "picture": "https://...",
    "disabled": false,
    "profile": "username",
    "customer": {
      "id": "cust-123",
      "name": "Kunde A"
    }
  },
  {
    "id": "ghi789-jkl012",
    "name": "Company LinkedIn",
    "platform": "linkedin-page",
    "picture": "https://...",
    "disabled": false,
    "profile": "Company Name",
    "customer": null
  }
]
```

### Antwortfelder

| Feld | Typ | Beschreibung |
|------|-----|--------------|
| `id` | string | Eindeutige Kanal-ID — verwenden Sie diese bei Post erstellen |
| `name` | string | Anzeigename des Kanals |
| `platform` | string | Plattform-Kennung (z.B. `instagram`, `linkedin-page`, `tiktok`, `x`) |
| `picture` | string | Profilbild-URL |
| `disabled` | boolean | Ob der Kanal deaktiviert ist |
| `profile` | string | Plattform-Benutzername oder Seitenname |
| `customer` | object/null | Kundengruppierung (falls zugewiesen) |

<Note>
Das Feld `id` ist der Wert, den Sie als `channel.id` beim Erstellen von Posts uebergeben. Das Feld `platform` gibt an, zu welchem Netzwerk dieser Kanal gehoert.
</Note>

## Einzelnen Kanal abrufen

```
GET /public/v1/channels/:id
```

Ruft die Metadaten eines bestimmten Kanals ab. Gibt `404` zurueck, wenn der Kanal nicht zu Ihrer Organisation gehoert.

```bash
curl -H "Authorization: YOUR_API_KEY" \
  https://api.postzee.app/public/v1/channels/abc123-def456
```

Das Antwortformat ist identisch mit einem einzelnen Eintrag im List-Endpoint.

## Legacy-Endpoint (veraltet)

```
GET /public/v1/integrations
```

Der vorherige Endpoint funktioniert weiterhin aus Kompatibilitaetsgruenden, ist aber als **veraltet** markiert und wird am **13.11.2026** entfernt. Migrieren Sie zu `/public/v1/channels`, sobald es Ihnen passt.

Beim Aufruf liefert der Legacy-Endpoint Deprecation-Header nach RFC 8594:

```
Deprecation: true
Sunset: Fri, 13 Nov 2026 00:00:00 GMT
Link: </public/v1/channels>; rel="successor-version"
Warning: 299 - "Deprecated API — use /public/v1/channels"
```

Der Antwortkoerper von `/integrations` verwendet weiterhin den Legacy-Feldnamen `identifier` (statt `platform`). Der neue `/channels`-Endpoint normalisiert dies zu `platform`.

Siehe den [Migrationsleitfaden](/de/api-reference/migration-v1-to-v2) fuer einen direkten Vergleich.
